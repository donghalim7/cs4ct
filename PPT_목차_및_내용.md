### 3. Our Challenges (우리의 도전)
**내용:**
- **문제 상황 1: 분산된 CS 관리**
  - 여러 회사(차널톡, 마신사, 솜성)와 부서에 걸친 CS 문의 관리의 복잡성
  - 각 부서별로 다른 관리 방식으로 인한 비효율성

- **문제 상황 2: 현황 파악의 어려움**
  - 부서별 CS 배정 현황을 한눈에 파악하기 어려움
  - 어떤 CS가 어느 부서에 배정되었는지 추적 어려움
  - 중복 배정된 CS 식별의 어려움

- **문제 상황 3: 데이터 분석의 비효율성**
  - CS 처리 현황 및 통계를 수동으로 집계해야 함
  - 시각화된 데이터 부재로 인한 의사결정 지연
  - 실시간 모니터링 시스템 부재

- **문제의 영향**
  - 고객 응대 시간 지연 → 고객 만족도 저하
  - 비효율적인 자원 배분 → 운영 비용 증가
  - 데이터 기반 의사결정 어려움

- **시각적 요소:** 문제점을 강조하는 아이콘, Before 상황 이미지

---

### 4. Our Solution (우리의 해결책)
**내용:**
- **통합 CS 관리 대시보드**
  - 모든 회사와 부서의 CS 현황을 하나의 대시보드에서 관리
  - 3단계 페이지 구조로 직관적인 네비게이션

- **실시간 모니터링 시스템**
  - 최신 CS 메시지 실시간 표시
  - 부서별 CS 큐 및 배정 현황 즉시 반영
  - Timestamp 기반 최신순 정렬

- **데이터 기반 분석 도구**
  - 다양한 통계 그래프 제공 (완료율 추이, 팀원별 배정, 카테고리별 분포 등)
  - KPI 카드를 통한 핵심 지표 한눈에 파악
  - 시각화를 통한 트렌드 분석

- **효율적인 검색 및 필터링**
  - 부서명 실시간 검색 기능
  - CS 중복 제거 로직으로 정확한 데이터 제공

- **기대 효과**
  - CS 처리 시간 단축 (30% 이상)
  - 운영 효율성 증대
  - 고객 만족도 향상
  - 데이터 기반 의사결정 지원

- **시각적 요소:** 시스템 전체 구조 다이어그램, After 상황 이미지

### 5. How We're Different (차별점)
**내용:**
- **1. 직관적인 UI/UX 디자인**
  - 민트색, 보라색, 검은색, 흰색을 활용한 모던한 디자인
  - 카드 기반 레이아웃으로 정보 구조화
  - 그라데이션, 그림자, 호버 효과로 시각적 피드백 제공
  - 가로 레이아웃으로 정보 밀도 향상 및 스크롤 최소화

- **2. 실시간 데이터 처리**
  - Supabase 실시간 기능 활용
  - Timestamp 기반 정확한 시간 순서 정렬
  - msg_id 기준 중복 제거로 데이터 정확성 보장

- **3. 확장 가능한 아키텍처**
  - 다중 회사 및 부서를 지원하는 유연한 데이터 모델
  - 모듈화된 코드 구조로 쉬운 기능 추가
  - Streamlit + Supabase 조합으로 빠른 개발 및 배포

- **4. 개발 효율성**
  - Python 기반으로 데이터 분석과 시각화 통합
  - BaaS(Supabase) 활용으로 백엔드 인프라 구축 최소화
  - 코드 기반 UI로 버전 관리 및 협업 용이

- **5. 사용자 중심 설계**
  - 검색 기능으로 빠른 부서 접근
  - 명확한 페이지 구조로 학습 곡선 최소화
  - 반응형 디자인으로 다양한 화면 크기 지원

- **시각적 요소:** 차별점을 강조하는 아이콘, Before/After 비교 이미지

### 6. Flow (사용자 흐름)
**내용:**
- **사용자 여정 (User Journey)**

  1. **회사 선택**
     - 차널톡, 마신사, 솜성 중 관리할 회사 선택
     - 각 회사 카드 클릭으로 해당 회사 대시보드 이동

  2. **회사 대시보드 탐색**
     - 왼쪽: 부서 목록 확인 및 검색
       - 부서명으로 실시간 검색
       - 부서 설명 확인
       - 대시보드 버튼으로 상세 페이지 이동
     - 오른쪽: 전체 CS 현황 확인
       - 최근 들어온 CS 메시지 리스트
       - CS가 가장 많이 배정된 부서 순위

  3. **부서 상세 대시보드**
     - 최근 배정된 CS 큐 테이블 확인 (ID, 내용, 배정 시간)
     - KPI 카드로 핵심 지표 확인 (총 배정, 완료, 완료율)
     - 다양한 통계 그래프로 트렌드 분석
       - CS 완료율 추이 (라인 차트)
       - 팀원별 배정 현황 (바 차트)
       - 카테고리별 통계 (파이 차트)
       - 월별 처리 현황 (바 차트)

- **핵심 기능 플로우**
  - 검색 → 필터링 → 상세 확인
  - 회사 선택 → 부서 선택 → CS 큐 확인

- **시각적 요소:** 
  - 사용자 플로우 다이어그램 (화살표로 연결)
  - 각 페이지의 주요 화면 스크린샷
  - 인터랙션 포인트 강조

### 7. Result (구현 결과)
**내용:**
- **구현된 주요 기능**

  1. **회사 선택 페이지**
     - 3개 회사 카드 그리드 레이아웃
     - 각 회사별 현황 보기 버튼
     - 깔끔한 카드 디자인

  2. **회사 대시보드**
     - 부서 목록 (검색 기능 포함)
       - 가로 레이아웃: 부서명/설명 왼쪽, 대시보드 버튼 오른쪽
       - 실시간 검색으로 빠른 부서 찾기
     - 최근 CS 리스트
       - CS ID, 내용 미리보기
       - 최신순 정렬
     - CS 배정 많은 부서 순위
       - 부서명과 배정 건수 표시

  3. **부서 상세 대시보드**
     - CS 큐 테이블
       - CS ID, 내용, 배정 시간 (timestamp 기반)
       - 최신순 정렬
     - KPI 카드 3개
       - 총 배정된 CS 수
       - 완료된 CS 수
       - 완료율 (%)
     - 통계 그래프 4개
       - 완료율 추이 (라인 차트)
       - 팀원별 배정 현황 (바 차트)
       - 카테고리별 통계 (파이 차트)
       - 월별 처리 현황 (바 차트)

- **기술적 성과**
  - 실시간 데이터 처리 구현
  - 중복 제거 로직으로 데이터 정확성 확보
  - Timestamp 기반 정렬로 최신 정보 우선 표시

- **시각적 요소:** 
  - 각 페이지의 고품질 스크린샷
  - 주요 기능 하이라이트
  - 통계 그래프 예시
